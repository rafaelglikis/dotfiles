# Ralph

Run exactly one ticket implementation iteration. The caller is responsible for invoking this skill repeatedly in fresh sessions.

Feature or tickets directory:

$ARGUMENTS

## Process

1. Resolve `scripts/next_ticket.py` relative to this skill's directory and run `python3 <script-path> "$ARGUMENTS"`. The script accepts either `.scratch/<feature-slug>` or `.scratch/<feature-slug>/tickets`, validates all ticket frontmatter and blocking edges, and selects the lowest-ID frontier ticket without reading ticket bodies.
2. Handle the script's JSON `result`:
   - `error`: report its errors, then stop with `RALPH_ERROR`.
   - `done`: report completion, then stop with `RALPH_DONE`.
   - `blocked`: report every returned unfinished ticket with its status and unfinished blockers, then stop with `RALPH_BLOCKED`.
   - `continue`: use the returned ticket. Do not implement more than one ticket.
3. Read the selected ticket's complete Markdown file.
4. Record the current `HEAD`, then invoke and follow the `implement` skill for the selected ticket path. If the current tool cannot invoke the skill, load its instructions or perform the same workflow directly. Do not weaken its readiness, testing, review, status-update, or commit requirements.
5. After implementation returns, re-read the selected ticket and inspect the repository state. Success requires all of the following:
   - The ticket status is `done`
   - `HEAD` differs from the recorded commit
   - The implementation commit includes the ticket's status update
6. If every success condition holds, report `RALPH_CONTINUE` with the completed ticket ID, title, and commit. Otherwise report `RALPH_ERROR` with the failed conditions and stop. Never retry within the same invocation.

The final marker must be exactly one of `RALPH_CONTINUE`, `RALPH_DONE`, `RALPH_BLOCKED`, or `RALPH_ERROR`, placed on its own final line so an external loop can decide whether to run another fresh session.
